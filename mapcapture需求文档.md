# 旅游路线规划应用 - 需求规格说明书 (V1.0)

## 1. 文档概述
本文档基于与产品发起人的多次沟通整理而成，完整定义了名为“旅游路线规划应用”（暂定名）的功能需求、交互细节、技术选型建议及成本考量。旨在为后续的设计、开发与测试提供唯一、明确的依据。

## 2. 产品定位
一款旅游路线规划工具，帮助用户通过地图点标记、多图上传、链接和备注，快速制定并管理多个独立的旅行行程。核心价值在于提供流畅的离线体验，无需注册登录，数据完全归用户所有。

## 3. 核心功能模块

### 3.1. 行程管理
用户进入应用后，首先看到的是所有已创建的行程。
- **列表展示**：采用卡片式布局展示行程。
- **新建行程**：用户可创建新行程，需输入**行程名称**，可选填**日期范围**。
- **行程封面**：
    - **默认规则**：自动取该行程内第一个标记点的首张图片作为封面。
    - **手动修改**：支持用户从手机相册选择图片替换默认封面。
- **编辑与删除**：支持对已有行程进行重命名和删除操作。

### 3.2. 地图标记（基于高德地图）
进入具体行程后，主视图为可交互地图。
- **地图源**：高德地图（优先考虑其在国内POI的准确性与路况信息）。
- **添加标记点**：点击地图任意空白位置，弹出信息录入表单。
- **自动地址填充**：通过高德逆地理编码功能，自动将点击位置的地址解析为文本，并填充为标记点的**默认标题**。用户可手动修改该标题。
- **标记点详情**：
    - **多图上传**：支持从相册选择多张图片或实时拍照上传。图片在详情页中以网格形式展示。
    - **链接**：支持输入参考链接（如攻略网址、预订页面），点击可跳转。
    - **备注**：支持多行文本输入，用于记录心得、预算、营业时间等信息。

### 3.3. 行程规划与复盘
这是制定路线的核心场景，提供了除地图外的第二种视图。
- **入口**：在行程地图页，设有切换按钮可进入本行程的**缩略图列表模式**。
- **双视图切换**：
    - **默认视图**：**卡片列表模式**。每个卡片展示该点的首张图片缩略图、地点名称及备注预览。
    - **辅助视图**：**网格缩略图模式**。以网格形式密集展示所有点的图片，方便快速浏览视觉资料。
    - **切换逻辑**：用户可一键在两个视图间自由切换。
- **拖拽排序**：
    - 在卡片或网格模式下，每个列表项右侧提供拖拽图标（≡ 三横线）。
    - 用户长按拖拽图标并上下移动，即可调整标记点的顺序。
    - 松开后，新的顺序**立即保存至本地**，形成每日或自定义的游览路线。
- **点击交互**：点击任意列表项，可跳转回该点的详情编辑页，或在地图上高亮定位该点。

### 3.4. 数据存储
- **存储策略**：**纯本地存储**。所有数据均保存在用户设备中，无需联网，无需注册登录。
    - **结构化数据**（行程信息、点坐标、备注、链接、排序序号）：使用本地数据库（如 SQLite）存储。
    - **图片文件**：直接保存至手机应用专属的存储目录，数据库中仅记录文件路径。
- **数据安全与迁移**（建议功能）：
    - 提供**一键导出**功能，将所有行程数据（数据库+图片）打包成一个文件。
    - 提供**一键导入**功能，用户可在新手机上恢复或迁移所有数据。

## 4. 交互细节确认
以下为与发起人确认过的关键交互细节：

| 确认项 | 最终决策 |
| :--- | :--- |
| **列表默认视图** | **卡片列表模式** |
| **列表视图切换** | **支持切换为网格缩略图模式** |
| **行程封面规则** | **默认取行程首个点的首图，支持手动修改** |
| 点标题生成 | 自动填充逆地理编码地址，支持手动编辑 |
| 多图支持 | 支持一个标记点上传多张图片 |
| 多行程支持 | 支持创建多个独立的行程 |
| 排序范围 | 每个行程内部的标记点独立排序 |
| 用户系统 | **不需要，P0级要求** |

## 5. 技术实现建议
### 5.1. 技术栈选型
- **跨平台开发**（推荐，一套代码覆盖iOS和安卓）：
    - **框架**：Flutter 或 React Native。
    - **地图插件**：高德地图官方提供 Flutter_amap 和 React-native-amap3d 插件。
- **原生开发**：
    - **iOS**：Swift + SwiftUI/UIKit + 高德iOS SDK。
    - **Android**：Kotlin + Jetpack Compose/XML + 高德Android SDK。
- **本地数据库**：
    - Flutter：Floor 或 Drift。
    - React Native：WatermelonDB 或 Realm。
    - 原生：Android 用 Room，iOS 用 Core Data。
- **图片处理**：使用各平台原生图片加载库处理本地文件即可。

### 5.2. 数据库核心表设计（示意）
- **行程表**：`行程ID`，`名称`，`封面图路径`，`创建时间`，`开始日期`，`结束日期`。
- **标记点表**：`点ID`，`所属行程ID`，`标题`，`地址`，`纬度`，`经度`，`备注`，`链接`，**`排序序号`**（用于拖拽排序），`创建时间`。
- **图片表**：`图片ID`，`所属点ID`，`图片文件路径`，`上传顺序`。

## 6. 成本考量与风险提示
### 6.1. 高德地图成本
- **开发与自用阶段**：使用高德开放平台的免费额度即可，**成本为 0**。
    - 地图显示（SDK）：永久免费。
    - 逆地理编码（核心功能）：每月5万次免费额度，个人自用完全足够。
- **上架商用阶段**：如果未来计划将应用上架至任何应用商店（如App Store、华为应用市场），**必须采购高德商用授权**。
    - **基础版商用授权**：市场价约 **50,000元/年**。
    - **原因**：应用商店审核时会要求提供《高德开放平台商用授权书》，否则应用无法上架。
- **重要说明**：应用本身是否盈利、是否免费，不影响商用授权的判定。只要上架公开分发，即视为商业用途。

### 6.2. 其他成本
- **开发者账号**：若需上架，需支付Apple开发者年费（99美元/年）或安卓应用商店的注册费（通常一次性的，价格不等）。
- **服务器成本**：**无**。本方案采用纯本地存储，无需云端服务器。
